E-Sphere E-Book

Building a Headless Decoupled E-commerce Platform with Laravel 11, React (Vite), and Gemini 2.5 Flash AI Chatbot Integration

Introduction

Modern e-commerce requires lightning-fast loading speeds, modular architecture, and intelligent systems that help visitors buy. The E-Sphere project implements these requirements by combining Laravel 11 as a backend REST API database driver, ReactJS (Vite) as a single-page frontend application, and Google's Gemini 2.5 Flash API as an integrated AI Shopping assistant.

This handbook walks through the complete codebase implementation, highlighting design decisions, database architectures, state management patterns, and step-by-step instructions to recreate, scale, or deploy this e-commerce platform.

1. Architecture & Design Patterns

E-Sphere is designed as a decoupled headless architecture. This means the frontend and backend are completely separate projects that communicate exclusively over secure HTTP API routes using JSON payloads.

Headless Architecture Advantage

  • Independent Scaling: The React frontend can be served via a CDN (like Vercel or Netlify) at minimal cost, while the Laravel API can be hosted on a cloud instance scaling dynamically.
  • Developer Specialization: Frontend developers can work exclusively with Vite, Tailwind CSS, and React, while backend developers focus on migrations, database queries, and business logic.
  • Enhanced Security: By using Sanctum authentication tokens and routing all AI processing through Laravel, the frontend never exposes internal database schemas or secret third-party API credentials (like the Google Gemini API Key).

Cross-Origin Resource Sharing (CORS)

Because the React SPA runs on a different port (typically localhost:5173) than the Laravel REST server (typically localhost:8000), CORS headers are set up in Laravel's configuration to accept, process, and return credentials (tokens, cookies) from the React origins.

2. Backend: Laravel REST API

The backend of E-Sphere uses Laravel 11, running on PHP 8.2+. It provides secure endpoints for catalogs, carts, orders, and checkout validations.

A. Routing Architecture

Routes are defined in the routes/api.php file. All routes in this file automatically receive the /api prefix and run stateless sessions via Sanctum middleware.

// routes/api.php
use App\Http\Controllers\Api\AuthController;
use App\Http\Controllers\Api\ProductController;
use App\Http\Controllers\Api\ChatbotController;

// Public Endpoints
Route::get('/products', [ProductController::class, 'index']);
Route::get('/products/{slug}', [ProductController::class, 'show']);
Route::post('/chatbot/message', [ChatbotController::class, 'handleChat']);

// Authenticated Customer Endpoints
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/profile', [AuthController::class, 'profile']);
    Route::get('/orders', [OrderController::class, 'index']);
    Route::post('/checkout/place-order', [CheckoutController::class, 'placeOrder']);
});

B. Cart Merging Logic

A key user experience feature of E-Sphere is Cart Syncing. When a guest visitor browses the site, their cart is stored in their browser's LocalStorage. Once they register or log in, React sends this local cart to the Laravel backend. Laravel merges these temporary guest items with the customer's permanent database cart, verifying warehouse inventory dynamically.

3. Frontend: React Vite SPA

The frontend is a single-page application built on Vite and React 19. It uses Redux Toolkit for global state management and Tailwind CSS for a premium dark-theme UI.

A. HTTP Request Wrapper

To communicate with the backend, React uses a centralized Axios client that intercepts every request to attach the Sanctum bearer token and track guest session tokens.

// frontend/src/utils/api.js
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:8000/api',
  headers: {
    'Content-Type': 'application/json',
    Accept: 'application/json',
  },
});

// Request interceptor to attach authentication token
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    
    // Add guest token if available
    let guestToken = localStorage.getItem('guest_token');
    if (!guestToken) {
      guestToken = Math.random().toString(36).substring(2, 15);
      localStorage.setItem('guest_token', guestToken);
    }
    config.headers['X-Guest-Token'] = guestToken;
    return config;
  },
  (error) => Promise.reject(error)
);

export default api;

4. AI Chatbot Integration

The AI Chatbot integration connects the React client with the Gemini 2.5 Flash API securely through Laravel, allowing for dynamic recommendations and live customer support.

A. Laravel Controller Logic

The ChatbotController takes care of compiling local database knowledge (products catalog & user order history) and injecting it as context into Gemini's system instructions. This ensures that the AI answers accurately without fabricating products or orders.

// backend/app/Http/Controllers/Api/ChatbotController.php
namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\Product;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Http;

class ChatbotController extends Controller
{
    public function handleChat(Request $request)
    {
        $message = $request->input('message');
        $history = $request->input('history', []);

        // 1. Fetch active products for catalog context
        $products = Product::where('is_active', true)
            ->select('id', 'name', 'slug', 'price', 'discount_price', 'rating', 'stock')
            ->limit(80)->get();

        $catalogContext = "Product Catalog:\n";
        foreach ($products as $p) {
            $price = $p->discount_price ?? $p->price;
            $catalogContext .= "- **{$p->name}** | Slug: `{$p->slug}` | Price: ₹{$price} | Stock: {$p->stock}\n";
        }

        // 2. Fetch authenticated user details (if logged in)
        $user = $request->user('sanctum');
        $userContext = $user ? "User: {$user->name}. Orders:\n" : "Guest User.\n";
        if ($user) {
            foreach ($user->orders()->latest()->limit(3)->get() as $order) {
                $userContext .= "- Order #{$order->order_number} | Status: {$order->status} | Total: ₹{$order->total}\n";
            }
        }

        // 3. System Instructions
        $systemInstructions = "You are a shopping assistant. Speak in Hinglish or English. Recommend products from catalog. Format links as: [Product Name](/product/slug). Do not make up products. Context:\n" . $catalogContext . "\n" . $userContext;

        // 4. API Request Payload mapping
        $contents = [];
        foreach ($history as $msg) {
            $contents[] = [
                'role' => $msg['sender'] === 'user' ? 'user' : 'model',
                'parts' => [['text' => $msg['text']]]
            ];
        }
        $contents[] = ['role' => 'user', 'parts' => [['text' => $message]]];

        // 5. Query Google Gemini API
        $response = Http::post("https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=" . env('GEMINI_API_KEY'), [
            'contents' => $contents,
            'systemInstruction' => ['parts' => [['text' => $systemInstructions]]]
        ]);

        return response()->json(['reply' => $response->json()['candidates'][0]['content']['parts'][0]['text']]);
    }
}

B. React Chat Widget & Route Parser

The frontend chatbot component parses markdown links dynamically and maps them to SPA routes using React Router, keeping transitions seamless.

// Inside Chatbot.jsx - relative link routing
const renderMessageContent = (text) => {
  const parts = [];
  const regex = /(\*\*.*?\*\*|\[.*?\]\(.*?\))/g;
  let match;
  let lastIndex = 0;

  while ((match = regex.exec(text)) !== null) {
    if (match.index > lastIndex) {
      parts.push(text.substring(lastIndex, match.index));
    }
    const token = match[0];
    if (token.startsWith('**')) {
      parts.push(<strong className="font-bold text-white">{token.slice(2, -2)}</strong>);
    } else if (token.startsWith('[')) {
      const closingBracket = token.indexOf(']');
      const linkText = token.slice(1, closingBracket);
      const url = token.slice(closingBracket + 2, -1);
      
      if (url.startsWith('/')) {
        parts.push(<Link to={url} className="text-primary-300 underline">{linkText}</Link>);
      } else {
        parts.push(<a href={url} target="_blank" className="text-primary-300 underline">{linkText}</a>);
      }
    }
    lastIndex = regex.lastIndex;
  }
  return parts;
};

5. Database Schema & Relationships

The structural relational integrity of the e-commerce database is handled through migrations in Laravel, creating strong connections between users, products, orders, and reviews.

Database Schema Diagram

  • Users (1) ➔ Addresses (N): Users can save multiple shipping/billing addresses.
  • Users (1) ➔ Orders (N): Customers can place multiple orders over time.
  • Orders (1) ➔ OrderItems (N) ➔ Products (1): Links orders to concrete catalog items, keeping records of purchase prices at checkout.
  • Products (1) ➔ Reviews (N): Holds approved client ratings and comments.

6. Full Project Setup & Run

To run the website on any environment (such as a newly cloned system), execute these installation and startup steps:

A. Clone & Backend Setup

# Clone repository
git clone https://github.com/sonuukush/laravelreactecom.git
cd laravelreactecom/backend

# Install PHP dependencies
composer install

# Setup env variables
copy .env.example .env

# Configure .env with your DB connection details and Gemini Key:
# DB_CONNECTION=mysql
# DB_PORT=3307
# DB_DATABASE=ecom_db
# GEMINI_API_KEY=your_key_here

# Generate APP Key
php artisan key:generate

# Migrate database & seed mock store items
php artisan migrate --seed

# Create filesystem storage link
php artisan storage:link

# Start backend server
php artisan serve

B. Frontend Setup

cd ../frontend

# Install node dependencies
npm install

# Start Vite dev server
npm run dev
